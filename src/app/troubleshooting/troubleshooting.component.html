Things I've learned about debugging

If you're not sure if data is correctly moving from component to component, try console logging it in the destination. If it's there, the problem is within the component. If it's not, it's probably a problem with the transfer.

Use Lint to rule out dumb typos, etc.

Don't try to build all the functionality at once. Build one piece at a time, check to make sure it's working, then commit so you have a good record and can go backwards if necessary.

When using StackOverflow to research error messages, remember that the initial post usually has the "bad" code and it shouldn't be used as a model.

When working with routers and passing data, remember this lesson and Observables: https://www.learnhowtoprogram.com/javascript/angular-extended/optional-subscribing-to-observables

If you think the error is being caused by a process not completing, try moving code around to see if the source of the error changes.

Check to see if problems might be caused by data becoming available to the page/app too late in the process. Timing is important.
